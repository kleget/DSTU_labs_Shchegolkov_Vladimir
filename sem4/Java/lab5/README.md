# Лабораторная 5: Java Server Faces

Работа сделана по файлу `sem4/java/all_labs/lab5.pdf`.

Тема: JSF, Facelets, составные компоненты, managed beans, валидация, шаблоны страниц, MVC и хранение данных через JPA.

## Быстрый запуск

Из папки лабораторной:

```powershell
cd sem4/java/lab5
.\run.ps1
```

Открыть в браузере:

```text
http://localhost:8085/lab5/
```

Если порт `8085` занят:

```powershell
.\run.ps1 8090
```

Остановить сервер: `Ctrl+C` в терминале.

Тестовые пользователи:

```text
manager / manager
client / client
```

## Что реализовано

### Задание 1

Создан составной JSF-компонент `namedinput`:

```text
server/src/main/webapp/resources/mytaglib/namedinput.xhtml
```

Он содержит:

- текстовую метку;
- поле ввода `h:inputText`;
- вывод ошибки через `h:message` красным цветом.

Использование:

```xhtml
<my:namedinput label="Имя" property="#{registerBean.name}"/>
```

Компонент поддерживает вложенные валидаторы JSF через `for="input"`.

### Задание 2

Созданы JSF-страницы авторизации и регистрации:

```text
server/src/main/webapp/index.xhtml
server/src/main/webapp/register.xhtml
```

Форма регистрации содержит поля:

- имя;
- e-mail;
- телефон;
- пароль;
- роль пользователя.

Проверки:

- имя и пароль обязательны;
- e-mail проверяется регулярным выражением;
- телефон проверяется регулярным выражением;
- сообщения об ошибках выводятся рядом с полями.

Данные пользователей хранятся в таблице `app_user`.

### Задание 3

Создан общий шаблон страницы:

```text
server/src/main/webapp/WEB-INF/templates/layout.xhtml
```

В шаблоне есть три части:

- верхняя часть: логотип, название фирмы, имя текущего пользователя;
- средняя часть: содержимое конкретной страницы;
- нижняя часть: контакты администрации.

Страница фирмы:

```text
server/src/main/webapp/welcome.xhtml
```

Логотип подключен как PNG:

```text
server/src/main/webapp/resources/images/logo.png
```

### Задание 4

Спроектировано web-приложение для проката автомобилей.

Роли:

- `MANAGER` - менеджер фирмы;
- `CLIENT` - клиент.

После входа пользователь попадает на свою страницу:

```text
http://localhost:8085/lab5/manager.xhtml
http://localhost:8085/lab5/client.xhtml
```

Менеджер может:

- просматривать свободные автомобили;
- добавлять автомобиль;
- изменять автомобиль;
- удалять свободный автомобиль;
- смотреть список текущих аренд.

Клиент может:

- смотреть свои текущие аренды;
- удалить автомобиль из списка арендованных, то есть завершить аренду;
- смотреть свободные автомобили;
- выбрать автомобиль;
- указать срок аренды в днях;
- увидеть общую сумму;
- нажать кнопку "Арендовать".

Архитектура MVC:

- Model: сущности `User`, `Car`, `Rental`, enum `Role`;
- Controller/backing beans: `AuthBean`, `RegisterBean`, `ManagerBean`, `ClientBean`;
- View: xhtml-страницы JSF;
- Data layer: `RentalService` и JPA.

### Задание 5

Реализовано долговременное хранение через JPA и Hibernate.

Настройка:

```text
server/src/main/resources/META-INF/persistence.xml
```

Используется встроенная база H2:

```text
jdbc:h2:./lab5db;DB_CLOSE_DELAY=-1
```

Таблицы создаются автоматически через Hibernate:

```xml
<property name="hibernate.hbm2ddl.auto" value="update"/>
```

При первом запуске добавляются тестовые данные: менеджер, клиент и несколько автомобилей.

## Как работает

Запрос пользователя обрабатывается `FacesServlet`. JSF читает xhtml-страницу, строит дерево компонентов, связывает поля формы со свойствами managed beans через EL-выражения вида `#{authBean.name}`.

При отправке формы JSF:

1. Восстанавливает состояние страницы.
2. Применяет значения из HTTP-запроса к компонентам.
3. Проверяет данные валидаторами.
4. Записывает корректные значения в managed bean.
5. Вызывает action-метод, например `authBean.login`.
6. Возвращает новую страницу или перерисовывает текущую.

`RentalService` открывает `EntityManager`, выполняет JPQL-запросы и сохраняет изменения в H2. Для каждой операции используется транзакция.

## Необходимая теория

JSF, Java Server Faces, это серверный web-фреймворк Java для построения пользовательского интерфейса из компонентов. Разработчик описывает страницу как набор JSF-тегов, а JSF преобразует ее в HTML.

Facelets - это xhtml-страницы JSF. Они поддерживают шаблоны, подключение страниц, составные компоненты и EL.

Managed bean - Java-класс, доступный из JSF-страниц по имени. В работе используются CDI-аннотации `@Named`, `@SessionScoped`, `@RequestScoped`, `@ApplicationScoped`.

EL, Expression Language, связывает страницу с Java-кодом. Например, `#{registerBean.email}` означает свойство `email` класса `RegisterBean`.

`h:form` создает HTML-форму и нужен для отправки JSF-компонентов на сервер. Без `h:form` `h:commandButton` не сможет вызвать action-метод.

`h:inputText` - поле ввода, `h:outputText` - вывод текста, `h:message` - сообщение об ошибке конкретного поля, `h:messages` - общие сообщения страницы.

Составной компонент - собственный тег, собранный из других JSF-компонентов. Он лежит в `resources/<library>` и подключается через пространство имен `http://xmlns.jcp.org/jsf/composite/<library>`.

Валидация JSF проверяет значения до вызова action-метода. В работе используются обязательные поля, `f:validateRegex`, `f:validateDoubleRange`, `f:validateLongRange`.

Facelets-шаблон строится через `ui:composition`, `ui:insert` и `ui:define`. Общие элементы страницы находятся в одном файле, а конкретные страницы подставляют свое содержимое.

MVC разделяет программу на модель, представление и контроллер. В этой работе xhtml - представление, managed beans - контроллеры/объекты-подложки, JPA-сущности и сервис - модель и уровень данных.

JPA - стандартный API Java для работы с реляционными данными как с объектами. Сущности помечаются `@Entity`, первичный ключ - `@Id`, связи - `@ManyToOne`. Hibernate выступает реализацией JPA.

## Как защитить

1. Запустить `.\run.ps1` и открыть `http://localhost:8085/lab5/`.
2. Показать `resources/mytaglib/namedinput.xhtml` и объяснить, что это составной компонент из задания 1.
3. Открыть регистрацию, ввести неверный e-mail или телефон и показать сообщения валидации.
4. Зарегистрировать пользователя или войти как `client / client`.
5. На странице клиента выбрать автомобиль, указать количество дней и показать расчет общей суммы.
6. Нажать "Арендовать" и показать, что машина появилась в списке арендованных.
7. Выйти и войти как `manager / manager`.
8. Показать список текущих аренд, добавление/изменение/удаление свободного автомобиля.
9. Открыть `layout.xhtml` и объяснить три части шаблона: header, content, footer.
10. Открыть `persistence.xml` и объяснить, что данные хранятся в H2 через JPA/Hibernate.

Коротко по контрольным вопросам:

- JSF строит web-интерфейс из серверных компонентов.
- Managed bean связывает страницу и Java-логику.
- `@ApplicationScoped` означает один объект на все приложение.
- `@SessionScoped` хранит состояние пользователя в HTTP-сессии.
- `@RequestScoped` живет один HTTP-запрос.
- `h:dataTable` выводит коллекцию строками таблицы.
- `h:panelGrid` раскладывает компоненты в сетку.
- `h:commandButton` отправляет JSF-форму и вызывает action.
- `h:button` делает обычный переход без вызова серверного action.
- Ошибки вывода поля показываются через `h:message`.
- Атрибут `rendered` включает или выключает отображение компонента.
