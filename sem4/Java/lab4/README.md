# Лабораторная 4: JDBC и JPA

Работа сделана по файлу `sem4/java/labs/lab4.pdf`.

Тема: работа Java web-приложения с базой данных через JDBC и JPA, автоматическое создание структуры БД, аутентификация, роли и MVC.

## Быстрый запуск

Из папки лабораторной:

```powershell
cd sem4/java/lab4
.\run.ps1
```

Или из корня репозитория:

```powershell
.\sem4\java\lab4\run.ps1
```

После запуска открыть:

```text
http://localhost:8080/lab4/
```

Если порт 8080 занят:

```powershell
.\run.ps1 8081
```

Сервер останавливается через `Ctrl+C` в терминале.

## Что реализовано

### Задание 1

Создано web-приложение для работы с товарами через JDBC.

URL:

```text
http://localhost:8080/lab4/jdbc/products
```

База данных содержит таблицы:

- `category` - категории товаров;
- `manufacturer` - производители;
- `warehouse` - склады;
- `product` - товары;
- `book` - книги для JPA-версии прошлой лабораторной;
- `app_user` - пользователи для JPA-части.

На странице JDBC можно:

- просматривать товары;
- фильтровать товары;
- добавлять товар;
- изменять товар прямо в таблице;
- удалять товар.

### Задание 2

Приложение само проверяет, создана ли структура БД. Если таблиц нет, оно создает их через JDBC.

Код авто-создания находится в:

```text
server/src/main/java/lab4/jdbc/DatabaseBootstrap.java
```

Параметры соединения вынесены в XML:

```text
server/src/main/webapp/WEB-INF/db-config.xml
```

По умолчанию используется встроенная СУБД H2:

```xml
<url>jdbc:h2:./lab4db;DB_CLOSE_DELAY=-1</url>
```

Это сделано специально, чтобы лабораторная запускалась одной командой без установки PostgreSQL. H2 подходит под формулировку "какую-нибудь СУБД". Если нужен PostgreSQL, надо добавить JDBC-драйвер в `pom.xml` и поменять XML-конфиг.

### Задание 3

Сделана JPA-версия информационной системы из лабораторной 3: библиотека книг. В lab3 книги хранились в XML, здесь они хранятся в базе данных через JPA.

URL:

```text
http://localhost:8080/lab4/jpa/books
```

Для входа:

```text
user / user
admin / admin
```

JPA-сущности:

```text
Book.java
Category.java
Manufacturer.java
Warehouse.java
Product.java
AppUser.java
```

Настройка persistence unit:

```text
server/src/main/resources/META-INF/persistence.xml
```

В качестве JPA-провайдера используется Hibernate. `EntityManagerFactory` создается динамически по настройкам из `db-config.xml`.

Дополнительно оставлена JPA-страница товаров:

```text
http://localhost:8080/lab4/jpa/products
```

Основная страница для заданий 3-5 - `jpa/books`, потому что она является переносом библиотеки из lab3 на JPA.

### Задание 4

Добавлена аутентификация пользователей и роли.

URL входа:

```text
http://localhost:8080/lab4/login
```

Роли:

- `USER` - может работать с книгами в JPA-версии;
- `ADMIN` - может управлять пользователями и настройками соединения с БД.

Админка:

```text
http://localhost:8080/lab4/admin
```

В админке можно:

- добавлять пользователей;
- менять логин, пароль и роль;
- удалять пользователей;
- менять JDBC driver, URL, user, password для подключения к БД.

Пароли не хранятся открытым текстом. В таблице `app_user` хранится SHA-256 хеш пароля.

### Задание 5

JPA-часть организована по MVC.

Model:

```text
server/src/main/java/lab4/model/
server/src/main/java/lab4/jpa/
server/src/main/java/lab4/auth/
```

Controller:

```text
server/src/main/java/lab4/servlet/
```

View:

```text
server/src/main/webapp/WEB-INF/jsp/
```

JSP лежат в `WEB-INF/jsp`, чтобы пользователь не открывал их напрямую. Запрос сначала попадает в сервлет-контроллер, затем сервлет передает данные в JSP через `request.setAttribute(...)`.

## Как работает

При первом обращении к JDBC или JPA-части приложение:

1. Читает `WEB-INF/db-config.xml`.
2. Загружает JDBC-драйвер.
3. Подключается к БД.
4. Проверяет наличие таблицы `product`.
5. Если таблиц нет, создает структуру БД.
6. Добавляет начальные товары, книги и пользователей.

JDBC-страница работает напрямую через `Connection`, `PreparedStatement` и `ResultSet`.

JPA-страница библиотеки работает через `EntityManager`: Java-классы-сущности отображаются на таблицы, а запросы выполняются через JPQL.

## Необходимая теория

### JDBC

JDBC, Java Database Connectivity, это стандартный API Java для работы с реляционными базами данных.

Основные элементы JDBC:

- Driver - драйвер конкретной СУБД;
- Connection - соединение с БД;
- Statement - объект для выполнения SQL;
- PreparedStatement - заранее подготовленный SQL-запрос с параметрами;
- ResultSet - результат SELECT-запроса.

В работе используется `PreparedStatement`, потому что он безопаснее обычной склейки SQL-строк и защищает от SQL-инъекций.

### JPA

JPA, Java Persistence API, это более высокий уровень работы с БД. Вместо ручного чтения строк из `ResultSet` создаются классы-сущности.

Сущность JPA:

- помечается `@Entity`;
- имеет первичный ключ `@Id`;
- обычно имеет конструктор без параметров;
- отображается на таблицу БД.

`EntityManager` управляет сущностями: сохраняет, ищет, изменяет и удаляет их. Для запросов используется JPQL, где указываются имена сущностей, а не таблиц.

### MVC

MVC разделяет приложение на три части:

- Model - данные и бизнес-логика;
- View - отображение данных пользователю;
- Controller - обработка HTTP-запроса.

В этой лабораторной контроллеры - это сервлеты, представления - JSP, модели - сущности, репозитории и сервисы.

### Аутентификация и роли

Аутентификация проверяет логин и пароль. После успешного входа данные пользователя сохраняются в HTTP-сессии.

Авторизация проверяет роль:

- `USER` работает с книгами;
- `ADMIN` управляет пользователями и настройками БД.

Пароль хешируется через SHA-256. Это лучше, чем хранить пароль открытым текстом.

1. Открыть `lab4.pdf` и сказать, что лабораторная про JDBC и JPA.
2. Запустить проект:

```powershell
cd sem4/java/lab4

```

3. Открыть:

```text
http://localhost:8080/lab4/
```

4. Показать JDBC-версию:

```text
http://localhost:8080/lab4/jdbc/products
```

На защите показать просмотр, добавление, изменение и удаление товара.

5. Показать файл:

```text
server/src/main/webapp/WEB-INF/db-config.xml
```

Сказать, что параметры соединения вынесены из Java-кода в XML.

6. Показать:

```text
DatabaseBootstrap.java
```

Сказать, что приложение само создает таблицы, если БД пустая.

7. Войти как пользователь:

```text
user / user
```

Показать JPA-страницу библиотеки:

```text
http://localhost:8080/lab4/jpa/books
```

8. Показать сущности:

```text
Book.java
```

Сказать, что `Book.java` - JPA Entity-класс, который отображается на таблицу `book`. Это перенос библиотеки из lab3 с XML-хранения на хранение в БД.

9. Войти как администратор:

```text
admin / admin
```

Показать управление пользователями и настройками БД.

10. Показать MVC:

```text
servlet/ - контроллеры
model/, jpa/, auth/ - модель
WEB-INF/jsp/ - представление
```

Короткая фраза для защиты:

> В заданиях 1-2 реализован CRUD товаров через JDBC с автоматическим созданием структуры БД и XML-конфигом подключения. В заданиях 3-5 библиотека из прошлой лабораторной перенесена на JPA: книги хранятся в таблице `book`, вход пользователей работает через роли USER и ADMIN, а код разделен по MVC: сервлеты обрабатывают запросы, модель работает с данными, JSP отображают страницы.
