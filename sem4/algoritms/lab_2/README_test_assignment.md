# Тестовое задание: бинарный поиск + топ-10 на реальном датасете

## Что выбрано
Использован реальный датасет **IMDb Top 1000 Movies and TV Shows** с Kaggle.

Оригинальный источник на Kaggle:
- https://www.kaggle.com/datasets/harshitshankhdhar/imdb-dataset-of-top-1000-movies-and-tv-shows

Размер датасета на Kaggle:
- `imdb_top_1000.csv`, около **438.1 KB**
- примерно **1000 строк**

## Что делает решение
Скрипт `test_assignment_imdb.py`:
1. загружает CSV;
2. сортирует записи по `Series_Title`;
3. выполняет **бинарный поиск** по названию фильма;
4. выводит **топ-10 фильмов по рейтингу IMDb**.

## Почему это удобно для защиты
- датасет реальный;
- датасет небольшой;
- бинарный поиск реализован вручную;
- топ-10 считается по понятному признаку — `IMDB_Rating`.

## Как запустить
```bash
python test_assignment_imdb.py
```

## Что можно сказать на защите
- Датасет — реальный, взят с Kaggle.
- Бинарный поиск работает только по **отсортированному** массиву.
- Поэтому сначала список фильмов сортируется по названию.
- Затем по названию выполняется поиск за `O(log n)`.
- Топ-10 выводится по полю `IMDB_Rating`.

## Если спросят, почему не искал по рейтингу бинарным поиском
Потому что бинарный поиск обычно удобнее показывать на **поиске конкретного элемента**, например фильма по названию.
Для поиска лучшей десятки логичнее использовать **сортировку по признаку**.
