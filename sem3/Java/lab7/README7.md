# Лабораторная 7 — Swing (минимальные примеры)

Код: `sem3/Java/lab7`. Каждый файл = отдельное задание. Запускать из `sem3/Java`: `java lab7.ClassName` (при необходимости `javac lab7/*.java`). Во всех классах есть `main`.

## Файлы и задачи
- `Task1LookAndFeel.java` (задание 1): `JFrame` с комбобоксом Look&Feel + кнопка/чек/радио. В центре `JTable` с языками. Выбор LAF вызывает `UIManager.setLookAndFeel` и `SwingUtilities.updateComponentTreeUI(this)`, чтобы окно перекрасилось.
- `Task2ControlPad.java` (задание 2): две панели управления (слайдер, спиннер, кнопки «Старт»/«Стоп»). Чистый UI-шаблон без логики.
- `Task3BallController.java` (задание 3): панель с шариками. `javax.swing.Timer` двигает шары и отражает от краёв. Управление: максимум шаров (`JSpinner`), «Добавить», «Пауза/Старт» (флаг `run`).
- `Task4FunctionPlotter.java` (задание 4): выбор интервала, функции из трёх вариантов и цвета линии. В `paintComponent` берётся 200 точек на отрезке и рисуется ломаная; осей нет, только рамка.
- `CustomComponents.java` (задания 5/6): два компонента.
  - `ThreeLabelPanel`: три `JLabel` (верх/центр/низ) с разными шрифтами/выравниванием; можно задать фон и тексты.
  - `SimpleDie`: квадрат с точками. Свойства: значение (1–6), цвет точек/фона, активность. По клику, если активен, `roll()` → новое значение → `repaint()`. Точки рисуются по маске 3×3.
  - `CustomComponentsDemo`: окно-пример, показывает оба компонента и кнопку включить/отключить реакцию кости.
- `Task8SimpleGame.java` (задание 8): упрощённая игра на двоих. Поле — массив `board` (очки), `repeat` (ходит ещё раз), `loseAll` (обнуление). Кнопки «Бросить»/«Сброс», лог в `JTextArea`. Ход: бросок своей кости → движение по полю → применение эффекта → при финише +50 и объявление победителя. Флаг `done` блокирует ходы до сброса.

## Кратко о решениях
- Один файл на задание — проще читать и запускать.
- Анимация через `javax.swing.Timer`, он безопасен для Swing и работает в EDT.
- Рисование только в `paintComponent` с вызовом `super.paintComponent(g)`, чтобы не ломать перерисовку.
- Переключение LAF требует `updateComponentTreeUI`, иначе элементы не обновятся.
- Функции в `Task4` заданы лямбдами (`DoubleUnaryOperator`), без отдельных классов.
- Кость (`SimpleDie`) хранит только состояние; точки выводятся по фиксированной матрице, минимум кода.
- Игра (`Task8`) без потоков и анимации — только обработка кликов и изменение чисел.
