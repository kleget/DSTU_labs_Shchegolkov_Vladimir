# Лабораторная 3: серверные страницы Java, JSP и MVC

Эта лабораторная сделана по `sem4/java/labs/lab3.pdf`.

Тема лабораторной: JSP, JavaBean, сервлеты, MVC, web-формы, работа с XML-файлом на сервере и простая игра через браузер.

## 1. Что реализовано

В лабораторной есть 5 заданий:

1. Информационная система с хранением данных в XML.
2. Та же система организована по архитектуре MVC.
3. JavaFX-клиент для информационной системы.
4. Игра с сервером в крестики-нолики 3x3.
5. Игра также организована по MVC.

Для простоты выбрана информационная система "домашняя библиотека книг".

Данные книг хранятся в XML-файле:

```text
server/src/main/webapp/WEB-INF/books.xml
```

Игра "крестики-нолики" хранит состояние в HTTP-сессии пользователя.

## 3. Сборка и запуск сервера

Самый простой запуск без ручного Tomcat:

```bash
mvn -f sem4/java/lab3/server/pom.xml jetty:run
```

Или из папки лабораторной:

```powershell
cd sem4/java/lab3
.\run.ps1
```

Для другого порта через скрипт:

```powershell
.\run.ps1 8081
```

После запуска открыть:

```text
http://localhost:8080/lab3/
```

Остановить сервер можно в терминале сочетанием `Ctrl+C`.

Важно: команда запуска не возвращает prompt, пока сервер работает. Это нормально. Нужно оставить терминал открытым и перейти в браузере по адресу выше.

Если в консоли Jetty пишет предупреждения вида `The XML schema ... could not be found`, это не ошибка запуска. Сервер все равно работает, если ниже есть строки `Started ... /lab3` или страница открывается в браузере.

Этот способ использует Maven Jetty plugin из `server/pom.xml`. Он поднимает локальный web-сервер, разворачивает приложение с контекстом `/lab3` и не требует вручную копировать WAR-файл в Tomcat.

Если порт `8080` занят, можно выбрать другой порт:

```bash
mvn -f sem4/java/lab3/server/pom.xml -Djetty.http.port=8081 jetty:run
```

Если сервер запущен не на `8080`, в JavaFX-клиенте нужно заменить порт в поле API.

Если нужен именно классический запуск через Tomcat, серверная часть собирается Maven как WAR-приложение:

```bash
cd sem4/java/lab3/server
mvn clean package
```

После сборки появится файл:

```text
target/lab3.war
```

Его нужно развернуть в Tomcat. Так как WAR называется `lab3.war`, контекст приложения будет:

```text
/lab3
```

После развертывания открыть:

```text
http://localhost:8080/lab3/
```

Важно: проект использует `jakarta.servlet-api` версии `6.0.0`, то есть рассчитан на Tomcat 10+.

## 4. Теория для защиты

### Что такое JSP

JSP, Java Server Pages, это серверные страницы Java. Они похожи на HTML-страницы, но внутри могут содержать специальные JSP-элементы для генерации динамического ответа.

При первом обращении JSP-страница преобразуется контейнером в обычный сервлет. Потом этот сервлет компилируется и обрабатывает запросы пользователя. Поэтому JSP можно объяснить как удобный способ писать HTML-представление, которое на сервере превращается в сервлет.

### Основные элементы JSP

Директива JSP задает настройки страницы:

```jsp
<%@ page contentType="text/html; charset=UTF-8" pageEncoding="UTF-8" %>
```

Директива импорта подключает Java-классы:

```jsp
<%@ page import="java.util.List,lab3.Book" %>
```

Скриптлет выполняет Java-код внутри JSP:

```jsp
<%
    String message = (String) request.getAttribute("message");
%>
```

Выражение выводит значение в HTML:

```jsp
<%= message %>
```

Объявление добавляет метод или поле в сгенерированный сервлет:

```jsp
<%!
    private String esc(String s) {
        return s == null ? "" : s;
    }
%>
```

В этой лабораторной JSP используется только как вид, то есть для отображения HTML.

### Встроенные объекты JSP

В JSP доступны встроенные объекты:

- `request` - текущий HTTP-запрос.
- `response` - HTTP-ответ.
- `session` - HTTP-сессия пользователя.
- `application` - контекст web-приложения.
- `out` - поток вывода HTML.
- `config` - конфигурация сервлета.
- `pageContext` - общий объект доступа к контекстам JSP.
- `page` - текущая JSP-страница.
- `exception` - ошибка, если JSP является страницей ошибки.

В проекте особенно важен `request`: сервлет кладет данные в атрибуты запроса, а JSP читает их и показывает пользователю.

### Что такое JavaBean

JavaBean - это обычный Java-класс, который:

- реализует `Serializable`;
- имеет конструктор без аргументов;
- имеет приватные поля;
- имеет getter/setter методы для свойств.

В лабораторной JavaBean-модели:

- `Book.java` - модель одной книги;
- `GameBean.java` - модель игры крестики-нолики.

### Что такое MVC

MVC разделяет приложение на три части:

- Model - данные и логика предметной области.
- View - отображение результата пользователю.
- Controller - обработка запроса и выбор действия.

В этой лабораторной:

- модель библиотеки: `Book.java`, `BookStore.java`, `books.xml`;
- контроллер библиотеки: `LibraryServlet.java`;
- вид библиотеки: `library.jsp`;
- модель игры: `GameBean.java`;
- контроллер игры: `GameServlet.java`;
- вид игры: `game.jsp`.

Такое разделение нужно, чтобы HTML не смешивался с основной логикой обработки запроса.

## 5. Задания 1-2: информационная система на XML и MVC

### Что сделано

Создана простая информационная система "библиотека книг". Пользователь через браузер может:

- посмотреть список книг;
- отфильтровать книги по жанру;
- добавить новую книгу;
- удалить книгу по ID.

Для добавления книги есть проверка регулярными выражениями. Она сделана и в HTML-форме, и на сервере в `BookStore`, поэтому неверные данные не попадут в XML даже через JavaFX/API:

- название: буквы, цифры, пробелы и простые знаки, 1-80 символов;
- автор: буквы, пробел, точка, апостроф или дефис, 2-60 символов;
- жанр: только буквы, пробел или дефис, 2-30 символов;
- год: число от 1500 до 2099.

Данные хранятся в XML:

```xml
<books>
    <book id="1" title="Java для начинающих" author="Иванов" genre="программирование" year="2022"/>
</books>
```

### Как работает

Браузер отправляет запрос на:

```text
/library
```

Запрос принимает `LibraryServlet`.

Если это GET-запрос, сервлет получает список книг и передает его в JSP:

```java
req.setAttribute("books", new BookStore(getServletContext()).list(genre));
req.getRequestDispatcher("/WEB-INF/jsp/library.jsp").forward(req, resp);
```

Если это POST-запрос, сервлет смотрит параметр `action`:

- `add` - добавить книгу;
- `delete` - удалить книгу.

Класс `BookStore` работает с XML-файлом через DOM:

- `list(...)` читает книги;
- `add(...)` добавляет новую книгу;
- `delete(...)` удаляет книгу.

JSP `library.jsp` только показывает HTML: таблицу книг и формы.

### Что показать на защите

Открыть:

```text
http://localhost:8080/lab3/library
```

Показать:

1. Таблицу книг.
2. Фильтр по жанру, например `программирование`.
3. Добавление новой книги.
4. Удаление книги по ID.
5. Файл `books.xml`, где реально хранятся данные.

### Что сказать преподавателю

Короткий текст:

> В заданиях 1-2 реализована информационная система домашней библиотеки. Модель данных хранится в XML-файле `books.xml`. Класс `Book` является JavaBean для одной книги, а `BookStore` отвечает за чтение и изменение XML через DOM. Контроллер `LibraryServlet` принимает GET и POST-запросы, обрабатывает фильтр, добавление и удаление. После обработки он передает данные в `library.jsp`, которая является представлением и формирует HTML-страницу. Таким образом система организована по MVC.

## 6. Задание 3: JavaFX-клиент

### Что сделано

Создан JavaFX-клиент:

```text
client_task3/LibraryClientFx.java
```

Он работает с сервером через простой API:

```text
http://localhost:8080/lab3/api/books
```

Клиент умеет:

- загрузить список книг;
- отфильтровать книги по жанру;
- добавить книгу;
- удалить выбранную книгу.

### Почему есть `/api/books`

JSP удобно показывать браузеру, но JavaFX-клиенту неудобно разбирать HTML. Поэтому добавлен минимальный сервлет `BooksApiServlet`, который возвращает данные как CSV:

```text

1;Java для начинающих;Иванов;программирование;2022
```

Это не усложняет лабораторную, а просто дает клиенту удобный машинный формат.

### Как запустить JavaFX-клиент

```bash

java  --module-path "C:\soft\javafx-sdk-21.0.10\lib" --add-modules javafx.controls,javafx.graphics LibraryClientFx
```

Путь к JavaFX SDK нужно заменить на свой.

Перед запуском клиента должен быть запущен Tomcat с приложением `lab3`.

## 7. Задания 4-5: крестики-нолики 3x3 и MVC

### Что сделано

Реализована игра "крестики-нолики" на поле 3x3.

Правила:

- человек ходит первым;
- человек играет `X`;
- сервер играет `O`;
- пользователь вводит номер строки и столбца;
- сервер после хода пользователя автоматически делает свой ход;
- при победе, поражении или ничьей выводится сообщение;
- состояние игры хранится в HTTP-сессии.

### Как работает

Пользователь открывает:

```text
http://localhost:8080/lab3/game
```

Контроллер:

```text
GameServlet.java
```

Модель:

```text
GameBean.java
```

Представление:

```text
game.jsp
```

`GameServlet` получает из сессии объект `GameBean`. Если его еще нет, создается новая игра:

```java
GameBean game = (GameBean) session.getAttribute("game");
```

При POST-запросе сервлет читает параметры:

- `row` - строка;
- `col` - столбец;
- `action=reset` - новая игра.

Затем он вызывает метод модели:

```java
game.playerMove(row, col);
```

Модель проверяет ход, ставит `X`, затем сервер ставит `O` в первую свободную клетку. После каждого хода проверяется победа или ничья.

JSP `game.jsp` получает объект игры и показывает таблицу 3x3.

### Что показать на защите

Открыть:

```text
http://localhost:8080/lab3/game
```

Показать:

1. Таблицу 3x3.
2. Ввод строки и столбца.
3. Ход пользователя.
4. Автоматический ход сервера.
5. Несколько ходов до завершения игры.
6. Сообщение о результате.
7. Кнопку `Новая игра`.

### Что сказать преподавателю

Короткий текст:

> В заданиях 4-5 реализована игра крестики-нолики по MVC. Модель `GameBean` хранит поле 3x3, признак завершения игры и сообщение для пользователя. Контроллер `GameServlet` получает ход пользователя из формы, вызывает модель и передает результат в JSP. Представление `game.jsp` рисует таблицу, форму ввода хода и сообщение о результате. Состояние игры хранится в `HttpSession`, поэтому у каждого пользователя своя партия.

## 9. Типовые вопросы преподавателя

### Почему JSP не лежат рядом с `index.html`, а находятся в `WEB-INF/jsp`?

Чтобы пользователь не открывал JSP напрямую. Запрос сначала должен попасть в сервлет-контроллер, контроллер подготовит данные и только потом перенаправит запрос на JSP.

### Зачем нужен `RequestDispatcher.forward`?

Он передает управление от сервлета к JSP внутри сервера. Браузер при этом не делает новый запрос, а JSP получает те же `request` и `response`, включая атрибуты, которые положил сервлет.

### Где здесь Model?

Для библиотеки модель - это `Book`, `BookStore` и XML-файл `books.xml`. Для игры модель - это `GameBean`.

### Где здесь View?

Для библиотеки - `library.jsp`. Для игры - `game.jsp`.

### Где здесь Controller?

Для библиотеки - `LibraryServlet`. Для игры - `GameServlet`.

### Почему данные книг хранятся в XML?

Так требует задание. XML содержит структурированные данные, а Java-код читает и изменяет его через DOM.

### Почему используется DOM?

DOM простой и встроен в Java. Он подходит для маленького XML-файла лабораторной: документ целиком загружается в память, затем можно читать элементы, добавлять и удалять узлы.

### Где хранится игра?

В HTTP-сессии:

```java
session.setAttribute("game", game);
```

Поэтому если два пользователя откроют игру из разных браузеров, у них будут разные партии.

### Как сервер выбирает ход в крестики-нолики?

Минимально: сервер ставит `O` в первую свободную клетку. Для лабораторной достаточно, потому что задание требует генерацию хода сервером, а не сильный искусственный интеллект.

### Почему JavaFX-клиент использует CSV?

Потому что CSV проще разобрать в программе. Браузер получает красивый HTML через JSP, а JavaFX-клиент получает те же данные в простом текстовом формате.
